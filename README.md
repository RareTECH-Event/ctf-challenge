# ハッキングに挑戦してみよう！

## 遊び方

1. 「Code」をクリックし、メニュー内の「Download ZIP」をクリック。(添付画像参照)

2. ダウンロードされた「ctf-challenge-main.zip」を解凍する。
3. ctf-challenge-mainにある `index.html` を、普段使っているブラウザ（Chrome や Edge など）にドラッグ＆ドロップ。
4. 画面が表示されたら、そこから探索開始です！

## フラグの形式
`FLAG{文字列}` という形式を探してください。

## 問題

### 問題①：HTML ファイルに隠された「開発者のメモ」を見つけよう

Webサイトの「表側」には見えていなくても、「裏側（ソースコード）」には大切な情報が書き残されていることがあります。
開発者がうっかり消し忘れてしまった「メモ書き」の中に、フラグが隠されています。ブラウザの機能を使って、サイトの裏側を覗いてみましょう！

#### ヒント：裏側を見る方法
画面上の何もないところで右クリックします。
「ページのソースを表示」 をクリックしてください。

### 問題②：ログファイルに隠された FLAG を調べよう
システムは、いつ・誰が・何をしたかを「ログ（記録）」という形で保存しています。
今回は、特別にそのログファイルの中身を見ることができるようです。大量の記録の中から、開発者がテスト用に置いたフラグを探し出してください。

#### ヒント：ファイルを探す方法
配布されたフォルダの中にある 「logs」 という名前のフォルダを開いてください。
その中にある 「system.log」 というファイルをダブルクリックして開きます。
たくさんの文字が並んでいますが、そのどこかに FLAG{...} が紛れ込んでいます。

※「system.log」はWordやPCに標準搭載されているメモ帳などで開くことが可能です。

### 問題③：特別な「合言葉」で、秘密のデータを取り出そう！
悪意のあるハッカーが、Webサイトのログイン画面をすり抜ける手口を再現してみましょう。
このサイトのログイン画面には、**「SQL インジェクション」**という有名なセキュリティの弱点（脆弱性）が隠されています。

この弱点を突くと、パスワードを知らなくても、ある「特別な文字列」を入力するだけでログインできてしまいます。サイトの裏側に隠された「フラグ」を見つけ出してください！

#### ヒント：ブラウザの「隠しメッセージ」を探す方法
ログイン画面で「右クリック」して「検証」 を選びます。
右側に表示される画面の上の方にある 「コンソール（Console）」 というタブをクリックしてください。
そこには、システム開発者がうっかり残してしまった 「攻撃に使える合言葉（ヒント）」 が表示されているはずです。
その合言葉を、ユーザー ID やパスワードの欄にコピー＆ペーストして、ログインボタンを押してみましょう。
